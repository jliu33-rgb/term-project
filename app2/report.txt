Application 2 – College Data Set

Data Structure Chosen:
Direct addressing table indexed by student ID (index = ID).

Justification:
Application 2 performs frequent searches by student ID, and it
searches/deletes IDs in the range 1–2000. With direct addressing,
each ID maps directly to an array index, eliminating collisions and probing.
This gives O(1) worst-case search, insertion, and deletion. This approach is
memory-efficient only when the keys limit is small and known in advance,
which would fit the case presented.

Worst-Case Time Complexity:

createDataSet:
O(1)
Initializes the presence array for universe size U, which is basically linear.

destroyDataSet:
O(1)
Frees allocated memory.

insertion:
O(1)
Direct index assignment with a presence check.

searchID:
O(1)
Direct presence check and constant-time access.

deletion:
O(1)
Clears presence flag in constant time.
